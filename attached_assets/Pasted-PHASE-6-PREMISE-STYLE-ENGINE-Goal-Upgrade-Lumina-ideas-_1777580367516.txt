PHASE 6 — PREMISE STYLE ENGINE

Goal:
Upgrade Lumina ideas from small relatable observations into simple, funny, viral-style premises.

Current issue:
The system is technically strong, but outputs still feel too small, literal, and template-like.

We want hooks closer to:
- “i ghosted my own to-do list”
- “my brain hates me after 11pm”
- “2am me is a genius. 9am me is useless”
- “the fridge knows i’m lying”

---

CORE IDEA

Add PremiseStyle as a new upstream layer.

PremiseStyle = comedic transformation rule.

It should decide HOW the idea becomes funny before hook templates are used.

---

NEW FLOW

ideaCoreType
→ scenario
→ HookIntent
→ PremiseStyle
→ hook generation
→ comedy compression
→ validation
→ pattern / filming guide

Templates become fallback/supporting material, not the primary source of humor.

---

DATA STRUCTURE

Create:

type PremiseStyle = {
  id: string
  label: string
  purpose: string
  transformLogic: string[]
  worksBestWith: string[]
  hookShapes: string[]
  examples: string[]
}

---

SELECTION RULES

Select PremiseStyle based on:

1. ideaCoreType / ideaCoreFamily compatibility
2. scenario compatibility
3. HookIntent compatibility
4. recent usage

Hard rules:
- no same PremiseStyle twice in one batch
- avoid recently used styles across regenerate history
- boost unused styles
- do not force a style if it does not fit the scenario

---

COMEDY COMPRESSION

After premise generation:

Hook must be:
- preferably 6–10 words
- maximum 12 words
- one clear joke
- simple
- instantly understandable

Reject if:
- too clever
- too long
- over-explained
- multiple ideas in one hook
- generic
- not funny
- literal/descriptive only

---

PREMISE QUALITY RULES

Boost hooks that include:
- self-roast
- absurd metaphor
- identity contrast
- emotional exaggeration
- unexpected object/personification
- specific behavior
- sharp contradiction

Penalize hooks that:
- merely describe the action
- sound like old templates
- feel safe/neutral
- lack comedic twist
- could fit any scenario

---

INTEGRATION RULES

Do not remove existing systems.

Keep:
- ideaCoreType
- scenarios
- HookIntent
- HookLanguageStyle
- template catalog
- pattern mapping layer
- validation rails
- Llama mutation fail-open behavior

PremiseStyle should influence hook generation before templates.

Templates should become fallback/support, not the main humor source.
