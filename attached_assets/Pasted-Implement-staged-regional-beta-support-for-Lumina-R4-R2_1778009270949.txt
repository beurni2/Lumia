====================================================
R3 — REGION-TAGGED ANCHOR CATALOG, STAGED CAREFULLY
====================================================

Goal:
Add a small, curated region-aware deterministic anchor baseline, without destabilizing the core engine.

This is medium risk. Keep it small and staged.

Do not add a huge regional corpus.
Do not replace western anchors.
Do not add abstract anchors that break scenario quality.
Do not add anchors that cannot produce concrete Film-This-Now instructions.

Use region-tagged anchors only when:
- the anchor is concrete enough to film
- the scenario can remain safe
- the action is compatible
- the anchor has a clear daily-life context
- the validator can still reason about it
- it does not rely on private real data

Start with a small curated set per region.

Examples of possible anchor categories, not mandatory exact items:

nigeria:
- WhatsApp group plan
- transport fare / commute plan
- data subscription / phone data
- market errand
- power/light interruption only if handled carefully
- school/work deadline
- food/home situation
- fake bank alert only with privacy warning

india:
- group chat plan
- commute/train/metro/traffic
- college assignment / exam notes
- hostel/roommate item
- food delivery cart
- family plan
- work deadline
- study materials

philippines:
- group chat plan
- commute/traffic
- food delivery
- phone data/load
- family plan
- school/work task
- roommate/home item
- friend/social plan

western:
- preserve current baseline.

Requirements:
- Each new anchor must have:
  - region tag
  - compatible topic lanes
  - safe verbs/actions
  - privacy/safety notes if needed
  - scenario compatibility check
- Do not let a new anchor flow into generic verb-object nonsense.
- Do not let a new anchor reduce recipe-loop yield without telemetry.
- Keep additions small enough to hand-review.
- If any anchor causes repeated rejection or weird output, remove or disable it.

QA for R3:

- awkward/stereotyped wording yes/no
- rejection/yield stats

Acceptance:
- western baseline remains stable
- each non-western region shows region-relevant details in at least 50–60% of ideas
- 0 hook/scenario mismatches
- 0 generic verb-object nonsense
- 0 impossible actions
- 0 safety failures
- 0 obvious fake/stereotyped regional phrasing
- at least 18/20 ideas per region are human-filmable
- tests pass
- TypeScript clean
