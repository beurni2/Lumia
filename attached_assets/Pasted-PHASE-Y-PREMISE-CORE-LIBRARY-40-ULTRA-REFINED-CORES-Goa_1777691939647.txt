PHASE Y — PREMISE CORE LIBRARY (40 ULTRA-REFINED CORES)

Goal:
Create a high-quality Premise Core Library that generates strong, funny, and aligned ideas.

This library becomes the PRIMARY source of idea generation:
PremiseCore → Premise → hook + whatToShow + howToFilm

This must eliminate:
- weak comedy
- hook/scenario mismatch
- repetitive idea loops

---

PART 1 — CORE STRUCTURE

Create a new module:

premiseCoreLibrary.ts

Define:

type PremiseCore = {
  id: string
  family: string

  mechanism: string
  tension: string
  description: string

  generatorRule: string

  compatibleDomains: string[]
  compatiblePremiseStyles: PremiseStyleId[]

  examples: string[]        // 2–3 examples (DO NOT reuse directly)
  antiCopyHints: string[]   // patterns that must NOT be repeated
}

Create EXACTLY 40 cores.

Organize into 8 families (5 cores each):

1. self_betrayal

---

PART 3 — QUALITY RULES (STRICT)

Each core must:

- represent a DISTINCT comedic idea engine
- not overlap with another core
- not be a reworded version of another core
- produce multiple unique ideas (not one pattern)

Each generatorRule must:

- describe how to generate the premise
- include contradiction or tension
- avoid generic phrasing

PART 5 — INTEGRATION INTO GENERATION

Modify idea generation pipeline:

NEW FLOW:

1. select PremiseCore (weighted by DefaultTasteProfile)
2. select domain/scenario
3. generate ONE premise using generatorRule
4. derive:
   - hook
   - whatToShow
   - howToFilm

STRICT RULE:
All outputs must come from SAME premise.

No independent generation allowed.

---

PART 6 — ALIGNMENT REQUIREMENT

Before returning:

Verify:
- hook expresses the premise
- whatToShow visually acts out the premise
- howToFilm executes the scenario

If mismatch:
→ regenerate

PART 7 — ANTI-LOOP INTEGRATION

Track across last 5 ideas:

- PremiseCore.id

- mechanism

- reject repeating same core too frequently
- heavily penalize repeating mechanism
- ensure at least 1 fresh core in each batch

- antiCopyHints from each core
- existing seed hook fingerprints

- generated hook matches example patterns
- near-duplicate structure detected
