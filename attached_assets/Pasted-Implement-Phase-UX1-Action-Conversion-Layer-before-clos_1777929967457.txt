Goal:
Shift the product experience from “Lumina gives ideas” to “Lumina helps me film this right now.”

The idea card and Film-This-Now screen should reduce filming hesitation, lower embarrassment, and make each generated idea feel immediately actionable.

Hard rules:
- No migrations unless absolutely necessary.
- No API breaking changes.
- Prefer deriving new UI labels from existing idea fields.
- Preserve route response shape if possible.
- Keep tests and TypeScript passing.
- Do not touch core ranking/scoring unless needed to expose already-computable metadata.
- No validator loosening.
- Avoid overbuilding.

Feature 1 — Make Film This Now the primary CTA
- Every idea card should have a prominent primary button:
  “Film This Now”
- It should be visually stronger than secondary actions.
- Secondary actions can remain if already present:
  Save
  Make another
  Not for me
  Details
- But Film This Now must clearly be the main next step.

Feature 2 — Add filming-confidence labels
Add lightweight labels to idea cards and Film-This-Now where derivable with high confidence:
- No face needed
- No talking required
- Text overlay works
- Can film alone
- Bedroom easy
- Desk easy
- Car easy
- No public filming
- Low cringe
- 10–20 sec shoot
- No props needed

Use existing fields where possible:
- setting
- shotPlan
- howToFilm
- whatToShow
- filmingGuide
- pattern
- format
- trigger/reaction
- voice/style metadata
- selectedSituations if helpful

If uncertain, do not show the label. Do not fabricate.

Feature 3 — Add action-conversion metadata if easy
If feasible without API breakage, compute or derive:
- estimatedShootTime
- difficultyLabel
- embarrassmentRisk: low/medium/high
- faceRequired: boolean/unknown
- voiceRequired: boolean/unknown
- publicFilmingRequired: boolean
- canFilmAlone: boolean
- wouldFilmToday: likely/unlikely/unknown

If API shape cannot change safely, derive these client-side from existing fields and report that limitation.

Feature 4 — Improve Film-This-Now screen layout
Make the screen practical and direct.

Recommended layout:
1. Hook / overlay line
2. Setup
3. Shot steps
4. Action / reaction beat
5. Caption
6. Time required
7. Difficulty
8. Confidence labels

Example structure:
Setup:
“Sit at your desk. Hold your phone like you just received bad news.”

Shot 1:
“Open with the hook.”

Shot 2:
“Show the object/app/scene.”

Shot 3:
“React to the reveal.”

Action:
“Slowly close the laptop like it hurt your feelings.”

Caption:
“My budget and I are not on speaking terms.”

Time:
“12 seconds”

Difficulty:
“Bedroom easy”

Labels:
No public filming · Can film alone · Text overlay works

Feature 5 — Rename generated batch area if appropriate
If current UI says generic “Ideas,” consider:

- “Today’s 3”

Only do this if it fits current UI without churn.

Feature 6 — Optional visible learning microcopy
If signal submission is already happening, optionally show tiny learning feedback after user action:
- “Noted — more dry deadpan.”
- “We’ll avoid ideas like this.”
- “More low-effort filming.”
- “Got it — more creator-life ideas.”

Keep this subtle. Do not overbuild.
