# Overview

Lumina is a pnpm monorepo project designed as a creator tool to enhance daily consistency for English-speaking micro-creators (1K–50K followers). It streamlines the content creation loop from onboarding and style profiling to idea generation, templated content creation, side-by-side review, and export. The Phase 1 MVP focuses on a rule-based Style Profile, a cloud-based Ideator powered by Claude Haiku 4.5 for generating video ideas with strict constraints, templated content creation, and a side-by-side review process. Lumina supports micro-creators in the US, UK, CA, AU, IN, PH, and NG markets, aiming to validate this core loop.

# System Architecture

Lumina is a pnpm workspace monorepo built with TypeScript.

**UI/UX Decisions:**
- The mobile application is developed using Expo (React Native).
- The onboarding process is a 3-step flow: region selection, first video import, and two additional imports.
- The post-export review screen reads as a finish line: a single final-video preview, a 3-item confidence strip, an optional 2-action Quick Boost (Smoother flow / Faster hook with Fix → Done ✓ micro-interaction), a friction-free "Made with Lumina" watermark toggle, a primary "Save & Post" CTA, and low-emphasis "Make another version" / "Back to ideas" secondary actions. No before/after comparison, analysis blocks, or technical warnings.
- The post-export flow extends Save & Post into a complete posting loop: a "Just save" secondary path (quieter "Saved ✓ / Post it later" copy with no auto-handoff); a "Save & Post" success block ("Saved ✓ / Ready to post / Takes ~10 seconds") that auto-opens a bottom-sheet platform picker (TikTok / Instagram / Snapchat / Copy only); an inline platform handoff card ("Almost there!" + 2-step guide + caption box + Copy caption / Copy & open {Platform} buttons); real deep links via `expo-linking` with web fallbacks (TikTok `snssdk1233://`, Instagram `instagram://library`, Snapchat `snapchat://`); clipboard writes via `expo-clipboard` on every platform tap; and a return-loop "Posted? / How did it go?" card that surfaces via `AppState`/`useFocusEffect` once the user comes back from a platform deep-link, replacing the two text buttons with Make another version / Back to ideas / View saved videos.
- A 5-question screen biases format distribution, tone, effort, privacy, and hook style based on user input.
- One-tap export to the gallery with an optional "Made with Lumina" watermark.

**Technical Implementations:**
- **Monorepo Structure:** Managed with pnpm workspaces.
- **Node.js:** Version 24.
- **API Framework:** Express 5 interacts with PostgreSQL via Drizzle ORM.
- **Database Conventions:** Additive migrations, versioned registry, immutable primary-key types.
- **LLM Integration:** Claude Haiku 4.5 via Replit AI Integrations for idea generation.
- **Per-Creator Format Distribution:** Rule-based system derives a target mix of four canonical patterns.
- **Variation Injection:** Hard rule in the ideator system prompt forces variation across scenario, setting, emotional spike, hook style, payoff type, and prop/action, preventing repetition within a batch.
- **Caption Craft:** Ideator system prompt enforces caption quality for generated ideas.
- **Lumina Evolution Engine:** Derives pattern-level bias from recent feedback and action signals, aggregating data over 60 days to guide idea generation.
- **Style Extraction:** Rule-based extraction using regex, keyword frequency, and scene-change detection.
- **Trends:** Static, manually refreshed JSON bundles per region.
- **Authentication:** Clerk.
- **Web QA Mode:** Flag-gated bypass for testing onboarding with a seeded demo creator.
- **Validation:** Zod for schema validation.
- **API Codegen:** Orval generates API hooks and Zod schemas from OpenAPI specifications.
- **Build System:** esbuild for CJS bundle creation.
- **Feature Flags:** Key features guarded by feature flags.
- **Quota Management:** Manages per-creator daily quotas and AI spend caps.
- **Job Queue:** Postgres job queue with partial unique index for deduplication.
- **Error Handling:** Structured error capture, in-process per-IP rate limiting, and Pino structured logging.
- **Consent:** Endpoints for user consent, data export, and deletion.

**Feature Specifications:**
- The Ideator generates video ideas with hard constraints: hook ≤ 3 seconds/8 words, shoot ≤ 30 minutes, and ≥50% low-effort ideas per batch.
- **Hybrid Ideator Pipeline (Layers 1/2/4 shipped):** Cost-reduction pipeline routing requests through a deterministic local pattern engine first (`lib/patternIdeator.ts` — 6 templates × ~20 safe scenarios × 5 hook-style phrasings, fully memory-biased), filtering candidates through a 6-axis 0–10 scorer with one rewrite attempt (`lib/ideaScorer.ts`), and only falling back to Claude `generateIdeas` when fewer than 3 local candidates pass the gate. Daily per-creator cache on `creators.last_idea_batch_json` + `last_idea_batch_date` (migration #20) serves same-day repeat requests for free; demo creators bypass cache writes. The orchestrator (`lib/hybridIdeator.ts`) exposes `{ ideas, source: cache|pattern|fallback|mixed, usedFallback, counts }` and the `/api/ideator/generate` route shape is unchanged. Llama 3.1 fallback and Llama 3.2 Vision are deferred to follow-up sessions.
  - **Within-batch diversity (HARD):** `diversifiedSelect` is a four-pass picker (strict → structure-unique → fresh-axis → rescue) gated at score≥8, fed by a cartesian-diagonal weave so every 16-candidate pool covers all 5 hookStyles, all 6 structures, and ≥3 distinct families.
  - **Between-batch regenerate variation (HARD, with rolling history):** When `regenerate=true`, the orchestrator reads a rolling cache history (the most-recent batch + up to `MAX_HISTORY_BATCHES=4` prior batches = 5 batches total) and applies a SPLIT HARD exclusion: every HOOK shipped in the last 5 batches is forbidden (depth 5 — mathematically prevents an identical 3-hook batch from re-appearing), but FAMILY/style exclusion only spans the immediately-prior batch (depth 1 — extending it to depth 5 squeezed the ~20-family pattern pool to ~6 candidates and starved the per-batch axis guards). Older-batch family/topic repetition is handled SOFTLY by the novelty-context penalties. The pattern engine is salted with `((hashEntries(flatHistory) ^ Date.now()) >>> 0) % 997` so each regenerate produces a structurally different (template, scenario, style) starting offset. Claude fallback (when triggered) is hook-excluded too. The cache JSONB envelope is now `{ version: 2, current: [{idea, family, templateId}], history: [batch[], ...] }` — backward-compatible reader (`tryParseCachedEnvelope`) accepts both legacy `Idea[]` and `[{idea, family, templateId}]` shapes. JSONB column means no migration needed for the shape extension.
  - **Novelty-aware selection (SOFT, BOOST + PENALTY):** The greedy selector replaces the four-pass `diversifiedSelect`. Each candidate carries `visualActionPattern` (11-value enum: `kitchen_contradiction`, `desk_avoidance`, `phone_scroll_freeze`, `bedroom_avoidance`, `outfit_check_cut`, `text_message_panic`, `face_reaction_deadpan`, `mirror_self_call_out`, `gym_no_show`, `car_avoidance`, `meal_prep_chaos`) and `topicLane` (5 values: `food_home`, `work_productivity`, `social_texting`, `body_fitness`, `daily_routine`) derived from `scenarioFamily` via `VISUAL_ACTION_BY_FAMILY` / `TOPIC_LANE_BY_FAMILY` lookup tables (`patternIdeator.ts`). `scoreNovelty` (`ideaScorer.ts`) awards 0–5 across 5 dims (hookStyle, scenario, structure, visualAction, topic) compared to the prior batch + the in-progress pick set, but ONLY when `qualityScore≥8` (no quality-rescue). `selectionPenalty` re-applies per-pick demerits with set-based saturation (-2 hookStyle, -3 family, -1 structure, -2 topic, -2 visualAction). The orchestrator uses `selectWithNovelty` (greedy by `quality + novelty + penalty`) with HARD batch guards (never 3× same hookStyle/format/structure, never >2 share family/visualAction/topic); `exhaustiveReselect` (capped at count≤5, top-`count*4` candidates) brute-forces a guard-passing combination if greedy fails. Claude fallback now triggers on EITHER `merged.length<3` OR guard failure; if guards still fail post-Claude, ships best-effort with `hybrid_ideator.guards_failed_shipping_best_effort` warn log. QA: 5 regens → 6/6 unique batches, 0 duplicate hooks, `usedFallback=false`, ≥2 distinct visual scenes + ≥2 distinct hookStyles per batch.
  - **Llama 3.1 hook-mutation layer (v2 quality-first, stateless):** Bolted on top of the now-frozen pattern engine — `lib/llamaHookMutator.ts` rewrites HOOK (and optionally caption) on the chosen 3-pick batch only AFTER local fallback merge in `hybridIdeator.ts`. Pattern engine still owns ALL structural work (scenarios, structure, filming, shot plan); Llama never touches them. Triggers (priority order): `regenerate`, `templated_openers` (2+ shared opener or family), `recent_history_similarity` (2+ overlap with novelty ctx), `all_pattern_batch`, `borderline_novelty` (hookImpact<2 + total≥6). `pickMutationTargets` caps per trigger then HARD-clamps to ≤5 (regen→top5, templated→top5, history→top3, all_pattern→top2, borderline→filter then top5). One `meta-llama/llama-3.1-8b-instruct` chat call via `@workspace/integrations-openrouter-ai` (45s timeout, temp 0.85, 800 max tokens, json_object, ≤3 hookOptions per slot enforced post-parse). Validators (in order): `too_long` (>12) → `too_short` (<3) → `banned_opener` (`pov:|when you|watching|reading|how to`) → `sensitive_content` (dm/bank/password/ssn/salary/venmo/etc) → `abstract_advice` (deserve/learning to/etc) → `incomplete_or_dangling` (validateHook for ≤10, light dangling-only check for 11-12) → `scene_drift` (must share content noun with original setting/family/sceneBeat) → `same_opener` (lookupHookOpener differs). Each surviving rewrite is fed back through `scoreIdea` and only replaces the original if `hookImpact > original` OR `personalFit > original` OR `total ≥ original + 1` (quality-first override) — first winning hookOption per slot wins, remainder skipped. Caption mutation runs independently per slot using the same +1 total or +1 captionStrength rule. On Llama timeout/network/parse failure the entire batch ships unchanged; the OpenRouter client is lazy-initialized via `getOpenRouterClient()` so missing env vars never crash boot. `source` ladder: `pattern` (no fallback, no mutation) → `mixed` (mutation selected at least one rewrite) → `fallback` (all-Claude). Telemetry logged to `hybrid_ideator.llama_mutation`: reason, candidatesSent, optionsReturned, mutationsSelected, costEstimateTokens, rejectedReasonCounts, errored. ZERO schema changes — purely in-memory; cache writes use the mutated batch so next-regen exclusion lists stay accurate. The mutator's `source: "llama_3_1"` flag also naturally suppresses templateId in `toCacheEntries` (which only sets it for `pattern_variation` source). QA: 45/45 unit + integration assertions passing (triggers, validators, end-to-end real Llama call); on real-batch end-to-end, Llama returned 5 valid hook options for a regenerate-triggered batch with 0 mutations selected because none scored strictly better than the strong pattern originals — quality-first override behaved exactly per spec.
  - **Llama cost-control + anti-abuse layer:** Per-creator usage gates layered on top of the hook-mutation pipeline. ZERO schema changes — reuses the existing `usage_counters(creator_id, day, kind, count)` composite-PK table with three new `kind` values (`idea_request` += `result.ideas.length` per generation, `regenerate_request` += 1 per regen attempt regardless of result-count, `llama_call` fire-and-forget +1 per mutator attempt). `lib/usageTracker.ts` owns the layer: DB-backed `incrementUsage` / `getUsageToday` for the daily counters; in-memory `Map<creatorId, number[]>` for the 2-min sliding Llama-call window; in-memory `Map<creatorId, lastTs>` for the rapid-regen cooldown. **Gates (priority order, demo creators bypass everything via the same `creator.isDemo` check the cache and quota paths already use):** (1) HARD LIMIT — route-layer, `regenerate=true && idea_request_count > 100` → 429 with `{ error: "rate_limit_take_a_break", message: "Take a break — come back later for fresh ideas" }`; quota is NOT consumed on this path. (2) RAPID-REGEN COOLDOWN — `acquireRegenSlot(creatorId)` is the atomic check-and-record primitive that returns 0 for the first tap and a randomized 3000-5000ms sleep for the second tap within `<3s`; closes the TOCTOU window two parallel taps would otherwise have with separate read+record calls. (3) THROTTLE — mutator-layer, `>3` Llama calls in the trailing 2-min window → skip mutation with `costControlSkipped: true, skipReason: "throttle_2min"`. The mutator refreshes `ctx.usage.llamaCallsLast2Min` from the live in-memory ring buffer at the top of `maybeMutateBatch` so parallel requests can't all pass against a stale snapshot the route handed them pre-await. (4) DAILY ADAPTATION — probabilistic skip with `Math.random()` priority `adaptation_40` (75% skip when `idea_request_count > 40`) > `adaptation_25` (50% skip when `> 25`). (5) FALLBACK — any cost-control skip ships pattern-engine output unchanged (existing graceful path, no new branches needed). **Mobile error mapping:** `studio.tsx` and `(tabs)/index.tsx` both inspect `ApiError.data` on 429 and surface the verbatim server `message` for `error === "rate_limit_take_a_break"` (rather than the generic `buildErrorMessage` "HTTP 429: …" prefix). **Telemetry:** `ideator.usage` (per request, INFO) logs `creatorId, creatorIsDemo, count, regenerate, cooldownAppliedMs, ideaRequestCountAfter, regenerateRequestCountAfter, llamaCallCountBefore, llamaCallsLast2Min`; `hybrid_ideator.llama_mutation` gains `costControlSkipped, skipReason, ideaRequestCountToday, llamaCallsLast2Min`; `ideator.hard_limit_blocked` and `ideator.cooldown_applied` log when those gates fire. **Deferred (architect-noted, bounded impact):** parallel-request DB-level race on the hard-limit check (would require advisory locks; in practice bounds overshoot to a few extra ideas before the next tap blocks). QA: 30/30 unit + DB-roundtrip assertions passing — gate priority/boundaries, demo bypass, throttle isolation across creators, atomic cooldown under simulated parallel taps, day-bucket UTC date.
  - **QA hardening (anti-clone, scene-specific copy, no broken hooks):** Layered on top of the novelty selector to fix the "different ideas that read like the same video filmed three ways" failure mode. (1) **Hook completeness validator** — `validateHook` rejects hooks ending in dangling words (`and`, `vs`, `with`, `to`, `the`, `my`, `me`, etc.), unsubstituted `${...}` leaks, ellipsis truncation, or word-count outside 3–10. The old `clampHookWords` chop-to-10 is gone; instead `pickValidatedPhrasing` walks all variants for the chosen style and `assembleCandidate` returns `null` if none ship — the generator silently skips that (template, scenario, style) triple. The `tryRewrite` rescue path (`ideaScorer.ts`) likewise iterates every other style + every variant and returns `null` if nothing validates, never truncating. (2) **HookOpener taxonomy + within-batch HARD guard** — new `HookOpener` enum (9 values: `i_just`, `i_really`, `me_saying`, `the_way_i`, `why_did_i`, `pov_check`, `tell_me_why`, `what_i_planned_vs`, `the_audacity`) tagged on each `HookPhrasingEntry` and resolvable from any hook string via `lookupHookOpener` regex table; `batchGuardsPass` HARD-caps each opener at 1/batch (so a batch can't ship three "I just realized…" hooks even when hookStyle differs). (3) **Setting expansion + within-batch HARD guard** — scenario pool grew to 25 entries spanning all 8 settings (`bed`/`couch`/`desk`/`bathroom`/`kitchen`/`car`/`outside`/`other`) — added new families (`skincare`, `mirror_pep_talk`, `walk`, `doom_scroll_car`, `closet_pile`); `batchGuardsPass` HARD-caps each `setting` at 1/batch so three kitchen scenes never ship together. (4) **Per-visualAction `howToFilm`** — `HOW_TO_FILM_BY_VISUAL_ACTION` lookup with 13 framing snippets keyed by `visualActionPattern` replaces the v1 boilerplate "Phone propped at chest height". `batchGuardsPass` also HARD-rejects byte-identical `howToFilm` strings within a batch as defense in depth. (5) **Scene-specific `whatToShow`** — new `sceneBeat` field on every `Scenario` (1–2 sentence physical moment, e.g. "You step in front of the mirror in the new outfit, hold the look for one beat, then turn away") drives `buildWhatToShow`, replacing the generic template. (6) **Cross-batch novelty extension** — `NoveltyContext` gained `recentHookOpeners` (derived from cached hook text via `lookupHookOpener`, zero cache-shape change) and `recentSettings` (from `idea.setting`); `scoreNovelty` is now 0–7 (added opener + setting axes); `selectionPenalty` adds -2 same-opener and -2 same-setting. **QA gate (5 regens, deleted post-pass):** every hook validated, no 2+ openers per batch, no 2+ settings per batch, no identical `howToFilm`, all `whatToShow` ≥20 chars + scene-specific, no v1 boilerplate substring, `source: "pattern"` and `usedFallback: false` on all 5 batches.
